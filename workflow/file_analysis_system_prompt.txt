# 文件解读：planner.py

## 核心功能

1. 多轮智能搜索
- 基于用户输入的 query（报告主题），通过 generate_research_queries_unified() 生成子查询（sub-queries）
- 最多进行 3 轮 迭代搜索，每轮最多 3 个子查询
- 自动判断已有信息是否充足，充足则提前终止循环

2. 网络内容抓取与筛选
- 通过 fetch_urls() 执行实际网络搜索
- 使用 judge_relevant_web_context_query_batch() 判断搜索结果与主题的相关性，过滤掉无关内容
- 使用 combine_same_result() 合并相同来源的搜索结果

3. 事实提取（Fact Extraction）
- 将搜索到的网页内容切分成 chunks，存入共享内容池（Context Manager）
- 根据 use_full_search 模式决定提取方式：
- 完整搜索模式：调用 summarize_web_context_for_page_section() 进行深度摘要
- 普通模式：调用 chunks_to_facts() 将 chunks 转化为结构化事实（facts）

4. 事实去重与质量过滤
- 本地去重：quick_deduplicate_facts() 基于规则快速去重
- 语义去重 + 相关性判断 + 冲突解决：当事实数量超过 5 条时，调用 LLM 进行语义级去重和冲突事实的裁决，保留最可靠的版本
- 最终输出高质量的事实集合

5. 结果输出
- 将最终的事实列表存入 ctx.session.state['init_web_search']，供后续的 Outline（大纲生成）等下游 Agent 使用

在整体流程中的位置

InitResearchAgent（初始研究）
        ↓ 输出事实集合
outline.py（大纲生成）
        ↓
section_plan.py → section_writer.py → reviewer → renderer

它是整个报告生成流水线的第一步，为后续的大纲规划（outline）和章节撰写（section writing）提供扎实的背景知识基础。

关键设计特点

- 流式事件驱动：通过 yield Event(...) 实时向前端推送搜索进度（生成的查询词、找到的 URL、提取的事实数等），用户体验好
- 防止过早满足：提供 search_context 指导模型"不是为了搜索而搜索"，而是确保信息的全面性和深度
- 模块化工具链：搜索、相关性判断、事实提取、去重等环节都由独立的工具函数完成，职责清晰

## 文件清单
| 文件 | 职责摘要 |
|------|---------|
| `combine_plan.py` | 计划集成代理：将各节网络搜索结果注入大纲并生成集成提示，解析模型输出更新章节与标题。 |
| `init_research.py` | 初始研究代理：基于用户查询生成子查询、抓取网页、抽取事实并去重，结果写入会话状态。 |
| `outline.py` | 初始大纲规划：构建提示、解析并标准化大纲JSON，循环结束后生成写作章节列表；含升级检查子代理。 |
| `planner.py` | 顺序组装入口：将研究初始化与提纲循环组合成一个顺序执行代理，对外提供统一规划接口。 |
| `replan.py` | 重规划辅助：解析重规划JSON、合并网络搜索上下文、规范化章节名、处理继承逻辑。 |
| `replan_loop.py` | 多轮重规划循环代理：协调内容/结构检测、大纲重组与验证，持续优化大纲和报告状态。 |
| `section_plan.py` | 章节计划生成与优化：通过生成→修订→检查反馈的循环，生成并更新章节级的详细计划。 |
| `section_search.py` | 章节网络搜索代理：针对特定章节异步收集、去重、过滤相关事实，产生进度事件并存储结果。 |
